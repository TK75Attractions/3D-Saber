# 3D Saber 譜面エディター 利用ガイド

このツールは、3D Saber 専用の新しい **Unity EditorWindow 版譜面エディター**です。
`Assets/Note-Recorder/` にある旧譜面エディターとは別物で、旧シーンを開いたり Play モードに入ったりする必要はありません。

## 起動方法

1. Unity Hub から、このリポジトリの `Assets/`、`Packages/`、`ProjectSettings/` があるフォルダーを開きます。
2. Unity のコンパイルが完了するまで待ちます。
3. Unity 上部メニューの **`3D Saber/譜面エディター`** を選びます。
4. 開いたウィンドウは、必要に応じて Unity の他のウィンドウと同じようにドッキングできます。

メニューが表示されない場合は、Unity Console にコンパイルエラーが出ていないか確認してください。

## 基本的な作業の流れ

1. 新しい譜面を作るか、既存の `chart.json` を読み込みます。
2. ヘッダーで保存先となる曲フォルダー名と難易度を選び、BPM、オフセット、拍子を設定します。
3. プロジェクトへ取り込んだ音源を選び、波形と再生位置を確認します。
4. Snap の細かさ、ノーツの色、種類を選びます。
5. 縦タイムラインでタイミングを決め、XY パッドで出現位置を決めます。
6. 必要に応じて方向、Long のカット回数などを調整します。
7. 保存後、本編の曲選択またはテストプレイで確認します。

作業を終える前や別の譜面を開く前には、未保存表示を確認して保存してください。

## 画面と操作

### 縦タイムライン

縦タイムラインは、曲の時間、拍、小節を縦方向に並べて表示します。添付イメージのように、前後のノーツを見ながら配置とタイミングを編集するための中心画面です。

- 太い線は小節、細い線は拍や Snap の区切りです。
- 波形は音の位置を探すための目安です。
- クリックした位置は、現在位置またはノーツ配置時刻になります。
- ノーツを選択すると、種類、色、方向、位置などを編集できます。
- 右クリックすると、その位置のノーツを削除できます。
- マウスホイールで前後へシークします。
- `Ctrl` または `Cmd` を押しながらホイールを回すと表示倍率を変更します。

### XY パッド

3D Saber のノーツには、タイミングだけでなく判定面上の X・Y 座標があります。XY パッドは、現在選択している時刻のノーツを左右・上下のどこへ出すか決める画面です。

- 横方向が X、縦方向が Y です。
- グリッドへ配置すると、本編で使用する座標へ変換されます。
- 同じタイミングでも異なる位置へ複数のノーツを置けます。
- タイムラインは主に時間、XY パッドは主に空間配置を確認するために使います。

### 色

| 色 | 本編での意味 |
|---|---|
| 赤 | 右手のセーバーで切るノーツ |
| 青 | 左手のセーバーで切るノーツ |
| 金 | どちらのセーバーでも切れる特別ノーツ |

既存譜面との互換用に色指定なしを扱う場合も、どちらのセーバーでも切れるノーツとして扱われます。

### ノーツの種類

| 種類 | 内容 |
|---|---|
| Tap | タイミングに合わせて1回切る通常ノーツ |
| Direction | 指定方向へ切るノーツ |
| Long | 同じノーツを複数回切るノーツ |

Direction は、上下左右と斜めを含む8方向から選びます。Long の長さは現行ゲームでは曲線や帯の長さではなく、主に必要カット回数 `count` として扱われます。

### Snap

Snap は、ノーツを拍の区切りへ揃える機能です。たとえば 1/4、1/8、1/16、1/32 のように、細かい値ほど短い間隔へ配置できます。

- 大まかな配置は粗い Snap、細かなリズムは細かい Snap が便利です。
- Snap を変更しても、すでに置いたノーツが自動で移動するとは限りません。
- 表示中の Snap 値が、その時点の配置とシークの単位です。

### 音源

音源を選ぶと、タイムライン上の波形、再生、一時停止、シークを使って譜面と曲を合わせられます。

- BPM は曲のテンポに合わせます。
- BPM を変更しても、本編の再生・判定で基準になる `time` は維持されます。`beat` は表示や編集を補助する値として、新しいBPMに合わせて更新されます。
- オフセットは曲と譜面全体のずれを補正します。
- 正のオフセットは、本編上でノーツを後ろへずらす方向です。
- 音源が鳴らない場合は、AudioClip の選択とインポート完了を確認してください。
- エディターで選んだプレビュー用AudioClipと、保存先フォルダーにある本編用 `audio.mp3` / `audio.ogg` / `audio.wav` が異なる場合は警告が表示されます。警告を確認し、意図した音源か確かめてから保存してください。

## Undo / Redo

配置、削除、移動、種類や色の変更は、Undo / Redo の対象です。大きな範囲を編集する前には、通常の保存も行ってください。

## ショートカット

| 操作 | ショートカット |
|---|---|
| 再生 / 一時停止 | `Space` |
| 保存 | `Ctrl+S` / `Cmd+S` |
| Undo | `Ctrl+Z` / `Cmd+Z` |
| Redo | `Ctrl+Y` / `Cmd+Y`（環境により `Ctrl+Shift+Z` / `Cmd+Shift+Z`） |
| 選択ノーツを削除 | `Delete` / `Backspace` |
| タイムライン上のノーツを削除 | 右クリック |
| Snap 単位でシーク | `←` / `→` |
| 1小節単位でシーク | `Shift+←` / `Shift+→` |
| 連続シーク | マウスホイール |
| タイムラインをズーム | `Ctrl+ホイール` / `Cmd+ホイール` |

入力欄へ文字を入力している間は、誤操作防止のため一部ショートカットが無効になります。ショートカットは今後調整される可能性があるため、実際のウィンドウ内表示がある場合はそちらを優先してください。

## 保存先と本編のファイル仕様

本編が読む曲データは、次のフォルダーへ保存します。

```text
Assets/StreamingAssets/Songs/<曲フォルダー名>/
├─ chart.json
├─ chart_easy.json
├─ chart_normal.json
├─ chart_hard.json
├─ audio.mp3、audio.ogg、または audio.wav
└─ cover.png（任意）
```

- `chart.json` は曲一覧へ表示するために必要な基本譜面です。
- 難易度別ファイルがある場合、本編は選択難易度の `chart_<難易度>.json` を優先します。
- 難易度別ファイルがない場合、本編は `chart.json` へフォールバックします。
- ウィンドウのヘッダーで選んだ曲フォルダー名と難易度が、そのまま保存先になります。
- 別の既存譜面がある保存先へ書き込もうとした場合は、上書き確認が表示されます。曲と難易度を確認してから確定してください。
- 難易度名は通常 `easy`、`normal`、`hard` を使用します。
- 音源は `audio.mp3`、`audio.ogg`、`audio.wav` のいずれかの名前にします。
- ジャケットを付ける場合は `cover.png` にします。
- 曲フォルダー名は本編内の曲IDとして使われるため、保存後に不用意に変更しないでください。

Normal を基本譜面として扱う場合は、`chart_normal.json` と `chart.json` を同じ内容にしておくと、本編の難易度フォールバックとも整合しやすくなります。保存後は目的の曲フォルダーに譜面と音源が揃っていることを確認してください。

既存ファイルを保存する前には、バックアップが `Library/3DSaberChartBackups/` に作成されます。このフォルダーはUnityプロジェクト内のローカル作業用領域なので、必要なバックアップは別の安全な場所にも保管してください。

## 旧 Note-Recorder との違い

- 旧版: `Assets/Note-Recorder/Scenes/SampleScene.unity` を開き、Play モードで使用します。
- 新版: Unity メニューから EditorWindow を開き、編集モードのまま使用します。
- 旧版の8×8現在位置表示ではなく、新版は縦タイムラインとXYパッドを組み合わせます。
- 新版は本編の赤・青・金、Tap・Direction・Longを直接意識した作りです。

旧 Note-Recorder のシーンや保存データを削除する必要はありません。両方が同じ本編形式の `chart.json` を扱う場合でも、編集中のファイルを同時に開いて上書きしないでください。

## 現時点のゲーム仕様上の制限

プロセカのような **曲線スライド、曲がるロングノーツ、ノーツ同士を線で結ぶスライド軌道** は、現時点の3D Saber本編の譜面仕様にはありません。

現在の本編が扱うのは、固定座標に出現する Tap、方向指定の Direction、複数回カットする Long です。見た目だけ曲線を追加しても本編は軌道を解釈できないため、曲線スライド対応にはエディターだけでなく、譜面JSON、ノーツ生成、表示、判定処理の拡張が必要です。

## 困ったとき

- 曲一覧に出ない: 曲フォルダー内に `chart.json` があるか確認します。
- 難易度が切り替わらない: `chart_easy.json` などの名前と保存先を確認します。
- 音源が鳴らない: `audio.mp3` / `audio.ogg` / `audio.wav` の名前と、選択中のAudioClipを確認します。
- ノーツが曲とずれる: BPMとオフセットを確認し、波形を見ながら先頭の拍を合わせます。
- 新しいメニューが出ない: Unity Console のコンパイルエラーを解消し、再コンパイルを待ちます。
